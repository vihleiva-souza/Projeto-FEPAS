⚠️ Importante

Na primeira validação de um dia, leva ~50-52 segundos (inclui download).

Nas próximas validações do mesmo dia, leva ~15-20 segundos (usa cache).

📋 Fluxo Passo a Passo

1️⃣

Cliente Seleciona e Clica

O que faz: Abre portal → Seleciona data → Preenche dados → Clica [VALIDAR]

Dados enviados: Data (20260717), Teste ID, DE11, DE41

2️⃣

API Recebe Request

O que faz: Endpoint POST valida parametros

Arquivo: app_homolog_web.py → rota /api/validar/{cnpj}/{produto_id}/cliente

3️⃣

Sistema Procura Log Local

O que faz: Verifica se LOGS de TESTE/01_QRCARDSE/aud_20260717.txt existe

Diretório: /var/data/LOGS de TESTE/01_QRCARDSE/ (Render Disk)

4️⃣

Log Não Existe? AUTO-COLETA! ⚡

O que faz: Dispara script coletor_audit_qr_http.py

Script: Faz download automático da URL pública

5️⃣

Download de Logs

Estratégia 1 (Preferida): Tenta aud_20260717.txt (consolidado)

Estratégia 2 (Fallback): Se não existir, lista e baixa múltiplos aud_20260717*.fps.txt

6️⃣

Consolidação em Arquivo Único

O que faz: Junta todos os .fps.txt em um arquivo só

Resultado: LOGS de TESTE/01_QRCARDSE/aud_20260717.txt (pronto)

7️⃣

Validação do Teste

O que faz: Abre log consolidado e busca transações

Filtros: DE11=123456, DE41=TERMINAL01, Teste=01

8️⃣

Análise de Estrutura e Campos

O que faz: Valida cada mensagem contra roteiro

Valida: MTI, Bits obrigatórios, Campos, Formatos

9️⃣

Resultado Final

O que retorna: Status (APROVADO/NEGADO), Protocolo, Evidência

Cliente vê: Resultado + Detalhes + PDF com evidência

✨ Características Principais

🤖 Automático

Cliente não precisa fazer nada. Sistema cuida de toda coleta, consolidação e validação automaticamente.

🚀 Rápido

Primeira validação: ~50s (inclui download). Próximas: ~15s (usa cache). Retry automático em falhas.

☁️ Render-Ready

Funciona 100% em Render Cloud. Sem dependências de Windows (listafps.exe). Apenas Python stdlib.

💾 Persistente

Logs salvos em Render Disk. Reutilizados em próximas validações do mesmo dia. Sem redownload.

🔄 Fallback Inteligente

Se arquivo consolidado não existe, automaticamente baixa múltiplos .fps.txt individuais. Transparente para usuário.

📝 Rastreável

Todos os downloads/consolidações geram logs em temp/qr_audit_http/saida/. Fácil debug.

🛠️ Script de Coleta (HTTP)

✅ Novo Script: coletor_audit_qr_http.py

Localização: scripts/tools/coletor_audit_qr_http.py

Recurso principal: Download via HTTP direto (sem listafps.exe)

Compatibilidade: Windows ✓ | Linux ✓ | macOS ✓ | Render ✓

Uso Local (Teste)
  • python scripts/tools/coletor_audit_qr_http.py → coleta do dia anterior
  • python scripts/tools/coletor_audit_qr_http.py --date 20260717 → data específica
  • python scripts/tools/coletor_audit_qr_http.py --date 20260717 --force → recoleta mesmo se já processado
Estratégia de Download
  • Passo 1: Tenta aud_YYYYMMDD.txt (consolidado)
  • Passo 2: Se não existir, lista aud_YYYYMMDD*.fps.txt (individuais)
  • Passo 3: Faz download paralelo (rápido)
  • Passo 4: Consolida em arquivo único com cabeçalho padrão

🚀 Deployment no Render

Arquivo: render.yaml
  • Configurar Render Disk para persistir LOGS de TESTE
  • Mount Path: /var/data
  • Variável: HOMOLOG_LOGS_DIR=/var/data/logs (opcional)
Dependências
  • ✓ Python 3.9+ (Render fornece por padrão)
  • ✓ Sem pacotes adicionais (script usa apenas stdlib)
  • ✗ Não precisa listafps.exe
  • ✗ Não precisa Wine/Proton

💡 Dica

Render detecta Python automaticamente. Apenas inclua requirements.txt com dependências do Flask se houver.
